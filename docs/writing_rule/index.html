    サマリ

    • 本ページはmarkdownで手順書を作成する際のライティングルールをまとめたものです
    • 基本的には本ルールに沿って、手順を作成するようにしてください

    ルール

    ■最初に手順名を記載する

    • フォーマット 
        # 手順書名

    • サンプル

      手順書名

    ■改版履歴を記述する

    • フォーマット

      |日付|更新内容|氏名| |:--------------|:-----------|:---------------| |2021/01/26|新規作成|otay@shiratma.com|

    • サンプル

      日付 更新内容 氏名
      2021/01/26 新規作成 otay@shiratama.com

    ■作業フローを記載する

    • フォーマット 
        1. 項目1
    1. 項目2
    1. 項目3

    • サンプル

      1. 対象サーバへssh
      2. uname -aを実行
      3. ・・・

    ■入力値は以下のフォーマットを利用する

    • フォーマット 
    　```bash
    入力値
　```
    • サンプル
    uname -a
cd test

    ■出力例は以下のフォーマットを利用する

    • フォーマット
    !!!!success
    出力例
    • サンプル

    Success

    Linux PC00001 4.19.128-microsoft-standard #1 SMP Tue Jun 23 12:58:10 UTC 2020 x86_64 x86_64 x86_64 GNU/Linux

    ■警告は以下のフォーマットを利用する

    • フォーマット 
    !!!warning
    出力例
    • サンプル

    Warning

    rm: it is dangerous to operate recursively on '/'
    rm: use --no-preserve-root to override this failsafe

    ■異常は以下のフォーマットを利用する

    • フォーマット
    !!!failure
    出力例
    • サンプル

    Failure

    rm: it is dangerous to operate recursively on '/'
    rm: use --no-preserve-root to override this failsafe

    ■補足は以下のフォーマットを利用する

    • フォーマット 
    !!!note
    補足内容
    • サンプル

    Note

    以下のように表示されたとしても影響はありません　など
    ※作業実施時に読み飛ばしても影響ない範囲で記載する
